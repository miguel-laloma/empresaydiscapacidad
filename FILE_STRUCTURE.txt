EXPLICACIÓN DE CARPETAS ESPECIALES
===================================

_layouts/
  └─ Archivos HTML que envuelven el contenido Markdown
     - default.html: estructura base (html, head, body con navbar+footer)
     - page.html: extiende default para páginas simples
     - post.html: extiende default para artículos con meta información

_includes/
  └─ Fragmentos HTML reutilizables incluidos en layouts
     - navbar.html: menú de navegación
     - footer.html: pie de página

_posts/
  └─ Artículos del blog (procesados automáticamente)
     Formato: YYYY-MM-DD-titulo-del-post.md
     Se ordena por fecha descendente

assets/
  └─ Archivos estáticos (CSS, JS, imágenes)
     Se copian tal cual al sitio generado


CONTENIDO DEL ARCHIVO CSS
==========================

assets/css/style.css contiene:
  • Variables CSS para colores, tipografía, espaciado
  • Estilos base (html, body, tipografía)
  • Navegación responsiva con dropdowns
  • Diseño hero para inicio
  • Estilos blog (tarjetas, metadatos)
  • Footer multcolumna
  • Responsive media queries (tablet, móvil)
  • Accesibilidad: focus-visible, prefers-reduced-motion
  • Soporte para modo oscuro (prefers-color-scheme: dark)


CONTENIDO DE FRONT MATTER MARKDOWN
===================================

Cada archivo .md contiene front matter YAML entre ---:

Ejemplo para página:
---
layout: page
title: "Título de la Página"
permalink: /ruta/pagina.html
description: "Meta descripción"
---

Ejemplo para post:
---
layout: post
title: "Título del Artículo"
date: 2026-04-15
author: "Nombre Autor"
excerpt: "Resumen breve para listados"
categories: Categoria1 Categoria2
---

====================

1. GitHub detecta cambios en main branch
2. Ejecuta Jekyll automáticamente:
   • Lee _config.yml
   • Procesa archivos .md con layouts
   • Transforma includes y variables
   • Genera sitio estático HTML/CSS
3. Publica en https://tu-usuario.github.io/empresa-discapacidad/
4. Disponible en ~1-5 minutos


PERSONALIZACIONES COMUNES
==========================

Cambiar colores:
  → Editar :root en assets/css/style.css

Cambiar título/descripción:
  → Editar _config.yml

Editar navegación:
  → Editar _includes/navbar.html

Cambiar footer:
  → Editar _includes/footer.html

Añadir nueva página:
  → Crear archivo .md en raíz o carpeta
  → Usar layout: page en front matter
  → Agregar enlace en navbar.html

Añadir post:
  → Crear _posts/YYYY-MM-DD-titulo.md
  → Usar layout: post
  → Se mostrará automáticamente en /blog/
